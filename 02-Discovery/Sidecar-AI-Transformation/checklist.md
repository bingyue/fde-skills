# Sidecar-AI-Transformation — 执行清单

## 准备

- [ ] 确认 AI 转型场景、目标用户、业务 Owner、AIBP、Sponsor
- [ ] 收集主流程图、核心系统清单、接口现状、已知改造排期
- [ ] 收集当前业务基线：耗时、成本、准确率、返工率、转化率或风险事件
- [ ] 识别主链路阻力：系统、流程、数据、组织、合规、商业
- [ ] 确认数据可用方式：只读、脱敏、快照、影子数据、API、文件导入
- [ ] 确认安全合规底线：敏感字段、数据不出域、审计、等保、行业监管
- [ ] 确认试点边界：用户范围、时间窗口、业务范围、失败回退方式
- [ ] 建立飞书/企业协同空间：会议日历、项目群、文档、看板、问题升级路径
- [ ] 拉齐判断标准：Sidecar 是风险隔离与证据收集，不是永久绕开主系统

## 适配性判断

- [ ] 完成「主链路改造阻力」评分（1-5 分）
- [ ] 完成「价值不确定性」评分（1-5 分）
- [ ] 完成「失败可回退性」评分（1-5 分）
- [ ] 输出结论：建议 Sidecar / 谨慎 Sidecar / 不建议 Sidecar
- [ ] 明确不适合 Sidecar 的高风险项，如自动资金操作、生产控制、强监管裁决
- [ ] 如果不建议 Sidecar，输出替代路径：主链路配置、流程改造、数据治理、人工增强

## 执行

### Scope：最小价值闭环

- [ ] 用 5 句话重述场景、阻力、价值假设、关键风险、当前阶段
- [ ] 定义最小闭环：输入 → AI 处理 → 人审/操作 → 反馈 → 指标
- [ ] 明确 PoC 只验证 1-3 个关键假设
- [ ] 明确停止条件：哪些指标不达标就终止或降级

### Isolate：隔离边界

- [ ] 系统隔离：Sidecar 不直接修改主系统结构
- [ ] 数据隔离：明确只读、脱敏、快照或影子数据策略
- [ ] 权限隔离：最小权限、服务账号、独立 RBAC、访问留痕
- [ ] 组织隔离：FDE + AIBP + 业务小组有独立推进节奏
- [ ] 风险隔离：失败时可回到原人工流程

### Design：架构与契约

- [ ] 划分主链路必须保留能力与 Sidecar 可承载能力
- [ ] 定义读接口、写接口、人审接口、回调接口和异常处理
- [ ] 定义数据同步频率：实时、准实时、批量、快照
- [ ] 设计反腐层：字段映射、状态映射、业务语义翻译
- [ ] 设计审计日志字段：用户、对象、动作、输入、输出、审批、时间、结果
- [ ] 设计失败回退：AI 错误、接口失败、权限拒绝、业务拒绝采纳
- [ ] 标注需要 API-Design-Review、RBAC-Audit 复核的点

### Evidence：价值证据

- [ ] 建立 Golden Dataset，AIBP 确认样本来源、数量和覆盖类型
- [ ] 采集业务基线，避免只用主观反馈判断价值
- [ ] 定义 AI 评估指标和业务指标
- [ ] 定义采纳漏斗：触达、试用、常用、依赖、嵌入
- [ ] 建立周度 Demo 节奏，必须展示失败样本和指标变化
- [ ] 使用 RICE / ICE 给候选 Sidecar 能力排序
- [ ] 输出商业案例：收益、成本、风险、Option Value、追加投入建议

### Convert：阶段决策

- [ ] 定义 PoC 进入 Beta 的条件
- [ ] 定义 Beta 进入生产的条件
- [ ] 定义合并回主链路的条件
- [ ] 定义长期 Sidecar 产品化运行的条件
- [ ] 定义退役或终止条件
- [ ] 准备下一次决策会的议程、证据包和决策请求

### Anchor：治理与运营

- [ ] 完成 RACI：业务 Owner、AIBP、FDE、IT、安全、运维、客户成功
- [ ] 确认预算归属：PoC、Beta、生产、运营分别由谁承担
- [ ] 建立问题升级机制：阻塞超过 2 个工作日升级 Sponsor 或项目负责人
- [ ] 建立飞书/企业协同看板：采纳、反馈、失败样本、变更通知
- [ ] 衔接 FDE-Adoption-Growth，制定种子用户、培训和运营节奏

### Retire：终局管理

- [ ] 写明合并回主链路的路线图与依赖
- [ ] 写明长期 Sidecar 的正式治理方式：权限、SLA、运维、成本、审计
- [ ] 写明退役条件和数据/权限清理方式
- [ ] 沉淀资产：场景卡、接口契约、评估集、Prompt、Runbook、案例复盘

## 验收

- [ ] Sidecar 适配性结论已由 FDE + AIBP 确认
- [ ] 阻力地图、干系人地图、RACI 完整
- [ ] 主链路 vs Sidecar 能力边界表完整
- [ ] 接口、数据同步、权限审计、失败回退设计完整
- [ ] PoC→Beta→生产→主链路决策 Stage-Gate 完整
- [ ] Golden Dataset 与业务基线可追溯
- [ ] 周度 Demo 记录包含成功样本、失败样本、指标变化和下一步假设
- [ ] 生产前已触发 RBAC-Audit、RAG-Evaluation、Private-Deployment-Gateway 或相关门禁
- [ ] 商业案例包含收益、成本、风险、Option Value 与下一阶段决策请求
- [ ] 明确合并、长期治理或退役条件，避免永久旁路

## 复盘

- [ ] 复盘 Sidecar 是否真正降低主链路阻力
- [ ] 复盘业务采纳是否来自真实流程嵌入，而不是项目推动
- [ ] 复盘 IT / 安全 / 合规是否从阻力方变成共创方
- [ ] 复盘哪些能力应合并主链路，哪些应沉淀为通用 Sidecar 服务
- [ ] 复盘哪些指标没有采集，导致商业案例不充分
- [ ] 复盘是否存在影子 IT、权限扩大化、数据复制失控风险
- [ ] 将可复用模板沉淀到 `10-Templates` 或项目资产库

## FDE 阶段门禁

- [ ] PoC：场景卡双签、只读/沙盒权限确认、最小价值闭环与停止条件明确
- [ ] Beta：采纳漏斗、周度 Demo、失败样本库、RACI、飞书/企业协同看板就绪
- [ ] 生产：RBAC、审计、人审、评估回归、Runbook、回退演练通过
- [ ] 主链路决策：合并 / 长期 Sidecar / 退役三选一有证据和责任人
- [ ] 信创/私有化（如适用）：数据不出域、模型网关、国产化组件、审计要求确认

## 阻塞升级

- [ ] 数据权限超过 2 个工作日无进展 → 升级 AIBP + IT Owner
- [ ] 自动写回涉及高风险结果 → 停止自动化，改为人审或只读建议
- [ ] 业务采纳低于预期 → 触发 FDE-Adoption-Growth 做采纳诊断
- [ ] 接口边界不清 → 触发 API-Design-Review
- [ ] 权限或审计不清 → 触发 RBAC-Audit
- [ ] 主链路合并争议大 → 触发 Executive-Communication-Framework 组织高层决策会
