# Sidecar-AI-Transformation — 评估标准

## 1. 通过门槛

全部「必达指标」达到 **≥80%**，且无 P0 风险未关闭，方可标记本 Skill 交付完成。

若进入 Beta 或生产阶段，必须额外满足：

- RBAC / 审计 / 人审 / 回退机制有书面证据
- AIBP 与业务 Owner 对价值指标和采纳指标完成确认
- IT / 安全 / 合规对生产门禁无 P0 阻塞

---

## 2. 指标矩阵

| 维度 | 权重 | 必达标准 | 失败信号 | 验收证据 |
| --- | ---: | --- | --- | --- |
| 适配性判断 | 15% | 明确建议 / 谨慎 / 不建议 Sidecar，理由可追溯 | 只因「快」就建议旁路 | 适配性评分、判断说明 |
| 阻力地图 | 15% | 系统、流程、数据、组织、合规、商业六类阻力完整 | 只列技术阻力 | 阻力地图、干系人地图 |
| 边界设计 | 20% | 主链路与 Sidecar 能力边界清晰 | 自动写回无门禁 | 能力边界表、接口契约 |
| 风险隔离 | 15% | 数据、权限、审计、回退、人审设计完整 | 无回退方案或审计字段 | 权限矩阵、日志字段、回退流程 |
| 价值证据 | 15% | 有基线、Golden Dataset、采纳漏斗与周度 Demo 机制 | 只有模型指标无业务指标 | 评估集、指标表、Demo 记录 |
| Stage-Gate | 10% | PoC/Beta/生产/主链路决策条件明确 | PoC 成功后直接生产 | 路线图、门禁表 |
| 商业案例 | 10% | 收益、成本、风险、Option Value 与决策请求完整 | ROI 口径不清 | 商业案例、决策会材料 |

## 3. 分项验收细则

### 3.1 适配性判断

- 明确说明为什么不直接改主链路
- 使用「改造阻力 × 价值不确定性 × 可回退性」判断是否适合 Sidecar
- 对强监管、生产控制、资金、库存等高风险场景给出谨慎或不建议结论
- 未知信息必须标注「待确认」，不得编造客户数据

### 3.2 阻力地图与干系人地图

- 至少覆盖系统、流程、数据、组织、合规、商业六类阻力
- 每个阻力项必须有来源角色、影响阶段、隔离策略和升级路径
- 干系人至少包含 Sponsor、业务 Owner、AIBP、IT、安全/合规、一线用户
- RACI 需明确 FDE 与 AIBP 的双负责人边界

### 3.3 Sidecar 边界设计

- 必须输出主链路保留能力和 Sidecar 承载能力
- 自动写回必须有额外门禁；默认优先只读、影子、人审、受控写回
- 接口契约需区分读、写、人审、回调、异常处理
- 数据同步需说明实时性、主数据来源、字段映射和一致性风险
- 反腐层需说明如何避免旧系统语义污染新能力

### 3.4 风险隔离与治理

- 权限必须最小化，服务账号和人工账号边界清楚
- 审计日志字段应覆盖用户、对象、动作、输入、输出、审批、时间、结果
- 失败回退必须覆盖 AI 错误、接口失败、权限拒绝、业务拒绝采纳
- 生产前必须有 RBAC、评估回归、人审策略、Runbook、回退演练
- 私有化/信创场景必须说明数据不出域、模型网关、国产化组件兼容

### 3.5 价值证据与采纳

- 业务基线和目标指标必须可采集
- Golden Dataset 需由 AIBP 或业务 Owner 认可
- 指标必须同时包含 AI 质量指标和业务结果指标
- 采纳漏斗至少覆盖触达、试用、常用、依赖、嵌入
- 周度 Demo 需展示成功样本、失败样本、指标变化、下一周假设

### 3.6 Stage-Gate 与终局

- PoC、Beta、生产、主链路决策每阶段都有进入条件和退出标准
- 明确停止 / 降级条件，避免无限试点
- 明确合并回主链路、长期 Sidecar、退役三类终局
- 长期 Sidecar 必须纳入正式治理：预算、运维、SLA、审计、Owner

## 4. 评分方式

### 4.1 总分计算

```text
总分 = Σ(维度得分 × 权重)
```

每个维度按 0-5 分评分：

- 5 分：完整、可执行、证据充分，可直接进入下一阶段
- 4 分：主体完整，有少量待确认项，不影响关键决策
- 3 分：方向正确，但缺少关键证据或责任人
- 2 分：只有概念性描述，无法指导交付
- 1 分：存在明显误判或缺失
- 0 分：未提供

### 4.2 通过标准

| 阶段 | 最低总分 | 额外条件 |
| --- | ---: | --- |
| PoC 准备 | 80 | 适配性判断、阻力地图、最小闭环完整 |
| PoC 结束 | 82 | 有基线、评估集、周度 Demo 证据 |
| Beta 准入 | 85 | 采纳漏斗、RACI、权限审计草案完整 |
| 生产准入 | 90 | RBAC、审计、人审、Runbook、回退演练通过 |
| 主链路决策 | 85 | 合并 / 长期治理 / 退役条件和责任人明确 |

## 5. 样例验收标准

### 样例 1：PoC 准备通过

```text
Given 项目阶段 = PoC 准备
When 执行 Sidecar-AI-Transformation 适配性判断
Then:
  - 输出建议 Sidecar / 谨慎 Sidecar / 不建议 Sidecar 的明确结论
  - 阻力地图覆盖六类阻力
  - 最小价值闭环和停止条件已定义
  - AIBP 与业务 Owner 确认场景卡草稿
  - 未出现自动写回或敏感数据越权设计
```

### 样例 2：Beta 准入通过

```text
Given 项目阶段 = PoC 结束
When 申请进入 Beta
Then:
  - PoC 指标达到双方确认的最低目标
  - 失败样本已进入 Golden Dataset
  - 采纳漏斗至少覆盖触达、试用、常用
  - IT/安全已评审权限和审计草案
  - 下一阶段用户范围、时间窗口、回退机制明确
```

### 样例 3：生产准入通过

```text
Given 项目阶段 = Beta 结束
When 申请生产上线
Then:
  - RBAC-Audit 相关项无 P0 阻塞
  - 审计日志可查询并能关联操作人、输入、输出和结果
  - 评估回归通过，失败样本有处置策略
  - 人审策略和自动写回边界经业务 Owner 确认
  - Runbook 与回退演练完成
```

### 样例 4：主链路决策通过

```text
Given Sidecar 已稳定运行 2-4 周
When 做主链路决策
Then:
  - 商业案例包含收益、成本、风险、Option Value
  - 明确选择：合并主链路 / 长期 Sidecar / 退役
  - 决策有 Sponsor 或业务 Owner 确认
  - 长期 Sidecar 不再处于影子状态，纳入正式预算和运维
```

## 6. 不通过处置

| 级别 | 条件 | 动作 |
| --- | --- | --- |
| P0 | 涉及敏感数据越权、自动写回高风险结果、无审计 | 立即停止扩大范围，降级为只读/人审，触发 RBAC-Audit |
| P0 | 无业务 Owner / AIBP 确认仍推进生产 | 暂停生产门禁，补齐 RACI 与验收责任 |
| P1 | 价值指标不可采集 | 补业务基线和埋点，延后 Beta |
| P1 | 采纳不足但模型指标达标 | 触发 FDE-Adoption-Growth，调整流程嵌入和培训 |
| P1 | 接口边界不清 | 触发 API-Design-Review，补接口契约和反腐层 |
| P2 | 文档不完整或决策项不清 | 2 个工作日内补齐报告和决策请求 |

---

## 7. 质量红线

- 不得把 Sidecar 描述为绕过 IT / 安全 / 合规的手段
- 不得默认 PoC 成功即可生产上线
- 不得在无审计、无回退、无人审条件下建议自动写回主系统
- 不得只评估模型准确率而忽略业务采纳和流程嵌入
- 不得缺失合并、长期治理或退役条件
- 不得编造客户未提供的业务基线、ROI 或权限状态
