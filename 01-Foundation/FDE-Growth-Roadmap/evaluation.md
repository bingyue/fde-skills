# FDE-Growth-Roadmap — 评估标准

## 1. 通过标准

- 总分 **≥80/100**
- 任何红线项均未触发
- 至少完成一个真实职业实验，而非只产出文档
- 计划中每个关键目标都有证据和复盘日期

本 Skill 的交付成功是“决策质量和行动证据提高”，不是保证 Offer、晋升或薪资结果。

---

## 2. 百分制评分

| 维度 | 权重 | 满分标准 |
| --- | ---: | --- |
| 问题与价值清晰度 | 15 | 主模式明确；价值排序、硬约束和反目标完整 |
| 岗位与市场现实 | 15 | 基于真实 JD/访谈；区分职责、门槛和偏好 |
| 路径与选择质量 | 10 | 至少三条可行路径，含代价、可逆性和退出选项 |
| 实验设计 | 15 | 关键假设均有低成本实验、证据和停止标准 |
| 能力与作品证据 | 15 | 项目以问题—行动—结果—复用表达，无编造 |
| 行动可执行性 | 10 | 30/60/90 计划具备日期、产出、验收和复盘 |
| 关系与组织资本 | 10 | 覆盖导师、赞助人、目标网络及困难对话 |
| 可持续性与风险 | 10 | 覆盖健康、现金流、生活约束、隐私与止损 |

### 评分解释

| 分数 | 结论 | 动作 |
| ---: | --- | --- |
| 90–100 | 可执行 | 进入实验与行动，按月复盘 |
| 80–89 | 基本可执行 | 先补最低分维度再启动 |
| 60–79 | 规划不完整 | 不做重大职业决定，补事实与实验 |
| <60 | 高风险 | 回到问题定义和现实校准 |

## 3. 红线项

出现任一项即不通过：

- 编造项目指标、经历、岗位、薪资或市场数据；
- 对 Offer、晋升、薪资作确定性承诺；
- 仅凭性格测试、年龄、学历或一次失败作职业定论；
- 建议用户在未评估现金流、健康和家庭约束时冲动裸辞；
- 作品集泄露客户数据、源代码、商业秘密或受限信息；
- 将持续心理或身体健康问题仅解释为意志力不足；
- 对骚扰、歧视、违法或安全风险只建议“忍耐”。

## 4. 分模式验收

### 4.1 入行辅导

| 必达项 | 验收证据 |
| --- | --- |
| 可迁移能力已识别 | 能力—证据矩阵 |
| 至少三种进入方式完成比较 | 路径决策表 |
| 完成岗位现实验证 | 3 次信息访谈或等价证据 |
| 完成 FDE 工作样本 | 案例、Demo、评估或交付复盘 |
| 90 天计划可执行 | 日历化行动与复盘点 |

### 4.2 求职辅导

| 必达项 | 验收证据 |
| --- | --- |
| 目标岗位聚类清晰 | 10–20 份 JD 分析 |
| 简历/作品集以技能证据表达 | 定制简历与案例索引 |
| 求职漏斗可追踪 | 公司、关系、投递、面试看板 |
| 面试故事覆盖关键场景 | 成功/失败/冲突/取舍/复用案例 |
| 完成双向尽调 | 面试问题与岗位风险记录 |

### 4.3 晋升辅导

| 必达项 | 验收证据 |
| --- | --- |
| 下一级标准已明确 | 书面标准或经理确认纪要 |
| 四象限证据完整 | Delivery/Impact/Leverage/Leadership |
| 根因不只归结为“做得不够” | 能力/可见度/关系/组织空间判断 |
| 晋升项目已设计 | 目标、范围、观察者、指标和时间 |
| 存在 Plan B | 内部替代机会或外部市场验证 |

### 4.4 生涯设计

| 必达项 | 验收证据 |
| --- | --- |
| 三条 Odyssey 路径差异明显 | 延续/相邻/替代路径图 |
| 生活方式纳入决策 | 价值、能量和硬约束清单 |
| 关键假设经过实验 | 访谈、跟岗、项目或体验记录 |
| 决策保留选择权 | 止损、缓冲和退出方案 |

### 4.5 职场困境

| 必达项 | 验收证据 |
| --- | --- |
| 事实与解释已分开 | 问题记录 |
| 根因拆解覆盖六类 | Issue Tree |
| 至少四类选项完成比较 | 修复/转岗/外部/恢复 |
| 困难对话可执行 | 对话脚本、底线、BATNA |
| 专业风险得到升级 | 医疗/心理/HR/法律等记录（如适用） |

## 5. 领先指标与结果指标

### 领先指标

- 每月高质量信息访谈数；
- 完成的职业实验数；
- 新增可验证项目证据数；
- 目标关系中的有效互动数；
- 定制投递和面试复盘完成率；
- 30/60/90 行动完成率；
- 精力和恢复状态趋势。

### 结果指标

- 获得目标职责的项目或岗位；
- 进入下一面试环节或获得 Offer；
- 晋升、职责扩大或内部转岗；
- 工作满意度、成长感和生活适配改善；
- 可迁移技能、作品集、网络和选择权增强。

结果指标受市场和组织影响，只用于复盘，不作为对辅导效果的单一归因。

## 6. 样例验收

```text
Given：一名 5 年后端工程师希望 90 天内验证是否转向 FDE
When：执行 GROWTH 全流程
Then：
  - 分析 ≥10 份目标 JD，并完成 ≥3 次从业者访谈
  - 建立直接 FDE、相邻 Solutions、内部转型三条路径
  - 完成 1 个端到端 AI 交付案例，包含评估和采纳设计
  - 形成 3 个“问题—行动—结果—复用”项目故事
  - 明确出差、客户互动和责任密度是否可接受
  - 产出 30/60/90 计划与停止条件
  - 总评分 ≥80，且无红线项
```

---

## 7. 90 天复评

复评不只检查“做了多少”，还要回答：

1. 哪个职业假设获得了新证据？
2. 哪条路径的吸引力或可行性发生变化？
3. 哪项能力从“自评”变成“可验证证据”？
4. 哪个市场或组织信号推翻了原计划？
5. 是否需要继续、调整、加码或止损？
