# FDE-Growth-Roadmap — 执行清单

## 1. 模式选择

- [ ] 本轮只选择一个主模式：入行 / 求职 / 晋升 / 生涯设计 / 职场困境
- [ ] 用一句话定义要解决的问题
- [ ] 明确时间窗口：30 天 / 90 天 / 1 年 / 3–5 年
- [ ] 明确“不解决什么”，防止一次规划所有人生问题

## 2. 基础材料

- [ ] 当前简历或 LinkedIn/个人主页
- [ ] 2–3 个项目案例与可验证证据
- [ ] 目标 JD 10–20 份或目标角色说明
- [ ] 最近绩效反馈、晋升标准或面试反馈
- [ ] 价值排序、能量账本、生活和财务硬约束
- [ ] 已完成 [FDE-Self-Assessment](../FDE-Self-Assessment/README.md)

## 3. GROWTH 生涯画布

### G — Grounding

- [ ] 写出前 5 项职业价值，并强制排序
- [ ] 记录高能量与持续消耗任务各 3 项
- [ ] 写出城市、家庭、现金流、出差等硬约束
- [ ] 明确至少 2 个反目标或不可接受代价

### R — Reality

- [ ] 按职责聚类目标岗位，而非只看职位名称
- [ ] 区分必须具备、可迁移、可后补、招聘偏好
- [ ] 每项能力至少对应一个证据或明确标为缺失
- [ ] 校准客户互动、编码、出差、压力和结果责任预期
- [ ] 标注信息来源日期，避免使用过时市场结论

### O — Options

- [ ] 已建立延续、相邻、替代三条路径
- [ ] 每条路径包含工作内容与生活方式
- [ ] 每条路径包含收益、代价和不确定性
- [ ] 每条路径包含退出选项，不把选择视为不可逆

### W — Work Experiments

- [ ] 每个关键假设都有 2–6 周验证实验
- [ ] 至少安排 3 次从业者信息访谈
- [ ] 至少有 1 个真实工作样本/项目实验
- [ ] 写清实验成本、证据和停止标准
- [ ] 实验不是“再上一门课”

### T — Track Evidence

- [ ] 项目按“问题—行动—结果—复用”整理
- [ ] 指标真实可解释；没有指标时不编造
- [ ] 明确个人贡献和团队贡献
- [ ] 至少展示一次失败、取舍或复盘
- [ ] 作品集已脱敏，不泄露客户数据和商业秘密

### H — Horizon Review

- [ ] 已输出 30/60/90 天动作
- [ ] 每个动作有负责人、日期和验收证据
- [ ] 已定义继续、调整、加码和止损条件
- [ ] 已预约 30 天检查与 90 天复盘

## 4. 入行专项

- [ ] 识别当前背景的可迁移能力
- [ ] 判断直接进入、相邻岗位或内部转型哪条更合理
- [ ] 已完成至少一个端到端 FDE 案例
- [ ] 案例覆盖发现、构建、评估、交付、采纳中的至少三项
- [ ] 已验证自己对客户互动、模糊度和责任密度的接受程度
- [ ] 未因年龄、学历或旧职位名称直接否定可行性

## 5. 求职专项

- [ ] 每个目标岗位都有 JD 证据矩阵
- [ ] 简历针对目标职责排序，不是通用版本海投
- [ ] LinkedIn/个人主页展示价值主张与 Featured 作品
- [ ] 已建立目标公司、联系人、访谈、投递和面试漏斗
- [ ] 每两周检查各环节转化率
- [ ] 准备 5 个项目故事：成功、失败、冲突、取舍、复用
- [ ] 准备业务 case、系统设计、AI 评估和生产化面试
- [ ] 已完成双向尽调问题清单
- [ ] 未承诺或预测必然获得 Offer

## 6. 晋升专项

- [ ] 已获得下一级书面或事实标准
- [ ] 已按 Delivery / Impact / Leverage / Leadership 整理证据
- [ ] 区分能力、可见度、关系和组织空间问题
- [ ] 与经理提前一个评审周期对齐差距
- [ ] 设计了一个高价值、高可见度的晋升项目
- [ ] 明确谁能观察、评审并证明结果
- [ ] 同时拥有导师和潜在赞助关系
- [ ] 准备内部 Plan B 与外部市场验证

## 7. 职场困境专项

- [ ] 把事实、感受和解释分开
- [ ] 从角色、能力、关系、资源、组织、健康六类拆解
- [ ] 关键判断有证据收集计划
- [ ] 已比较修复、转岗、外部机会和暂停恢复
- [ ] 已准备困难对话的目标、底线和备选方案
- [ ] 未在高度情绪化状态下做不可逆决定
- [ ] 涉及健康、骚扰、歧视、违法或安全时已寻求专业支持

## 8. 质量与伦理

- [ ] 不编造用户经历、市场数据、岗位或薪酬
- [ ] 不把性格测评当作职业结论
- [ ] 不以单次面试、项目或反馈断言长期适配
- [ ] 所有建议符合用户现实约束
- [ ] 个人隐私、客户资料和面试材料已脱敏
- [ ] 使用 `evaluation.md` 评分达到 80 分，且无红线项
