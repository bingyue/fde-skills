# FDE Self Assessment

Use this skill when the user needs a structured FDE capability assessment, team capability review, or development roadmap.

## When To Use

- The user wants to evaluate whether they or a team member is ready for an FDE role.
- A PoC, Beta, or production delivery exposed capability gaps that need diagnosis.
- A team needs layered FDE training, mentoring, or role calibration.
- The user needs a 30/60/90 plan tied to concrete project deliverables.
- The user asks for pre-sales FDE capability evaluation or transition planning.

## How To Use

1. Read `README.md` for the four-role framework and execution steps.
2. Use `prompt.md` to generate the assessment, preparation checklist, acceptance review, or retrospective.
3. Use `checklist.md` to verify required evidence and deliverables.
4. Use `evaluation.md` to score output quality and identify gaps.

## Output Expectations

Produce Markdown with:

- Five-sentence problem restatement.
- Four-role scorecard: business consulting, architecture ownership, AI tuning, delivery operations.
- Evidence-backed level assessment, preferably L1-L4.
- Blocking weaknesses and their project impact.
- Skill-path recommendations for each gap.
- 30/60/90 improvement roadmap with deliverables, review cadence, and mentor or stakeholder checkpoints.

Do not rate capability based only on tools learned. Use delivered scenarios, customer evidence, artifacts, and review outcomes where possible.
