# FDE-PoC-Tech-Stack-Selector — Prompt 模板

## 1. 完整执行 Prompt

```text
你是一名国内企业 AI 项目的资深 FDE 与解决方案架构师。
请执行 FDE-PoC-Tech-Stack-Selector。

## 场景输入
- 客户/行业：{填写}
- 业务问题：{填写}
- 目标用户与工作流：{填写}
- PoC 核心假设：{若无，请协助生成}
- Primary Metric：{填写}
- PoC 周期：{如 10 个工作日}
- 团队：{人数、语言、AI/运维能力}
- 数据：{格式、规模、质量、敏感等级}
- 部署：{公有云/私有化/离线/混合云}
- 算力：{无/CPU/NVIDIA/昇腾/其他}
- 信创/等保/数据出域：{填写}
- 必须集成：{系统、API、SSO、飞书等}
- 并发/时延/预算：{填写}
- 当前候选或偏好：{可空}
- Beta/生产预期：{填写}

## 强制执行规则
1. 先判断输入是否足以选型；不足时输出待确认清单，不得编造。
2. 必须使用联网搜索，记录当前日期。
3. 至少检索 8 个来源，其中：
   - 至少 4 个官方文档、官方仓库或 Release；
   - 至少 3 个国内官方来源或国内实践；
   - “最新”信息优先 180 天内，基础协议文档除外。
4. 强制检索：
   - 当前国内可用模型/API及数据策略
   - 开源低代码/RAG/Agent 平台
   - Agent/MCP/编排框架
   - 评估与可观测工具
   - 模型推理与国产算力适配
   - Vibe Coding IDE/插件/CLI 与企业数据政策
   - 同类国内落地案例
5. 关键候选须核验版本/发布日期、许可证、维护状态、部署方式。
6. 如果无法联网，停止最终推荐，只输出“预选方案，待联网验证”。

## STACK-POC
S：将需求压缩为一个可证伪 PoC 假设。
T：完成最新技术检索证据表。
A：判断 L1 开箱即用 / L2 Vibe Coding / L3 工程化路径。
C：先做硬门禁，再按 100 分矩阵比较候选。
K：输出 10 日构建蓝图与 Go/No-Go。

## 输出要求
必须明确给出：
1. 唯一主方案：逐层写具体技术与选择理由；
2. 一个降级方案：触发条件与切换时限；
3. 至少两个淘汰方案及淘汰原因；
4. PoC→Beta/生产升级路径；
5. 模型、编排、RAG、数据、UI/API、评估、部署、Vibe Coding 全栈；
6. Mermaid 架构图；
7. 10 个工作日任务表；
8. Golden Dataset、Primary Metric、Guardrail 与 Go/No-Go；
9. 成本区间、风险、回退和待确认项；
10. ADR 技术决策记录。

不得：
- 只列工具不做决策；
- 根据 Star 数直接推荐；
- 假装知道最新版本；
- 为 PoC 提前建设完整生产平台；
- 推荐未经批准的 AI 编码工具接触客户数据和密钥。
```

## 2. 快速 48 小时选型 Prompt

```text
基于以下场景，在 48 小时内完成 PoC 技术选型：
{粘贴场景卡、SOW、PRD 或会议纪要}

第一轮：
- 列出最多 8 个阻塞选型的关键问题；
- 对明确的信息不重复提问；
- 判断是否需要数据/文档 bake-off。

第二轮（必须联网）：
- 检索最新国内 AI 技术和官方资料；
- 只保留最多 3 个候选方案；
- 先做硬门禁，再评分。

第三轮：
- 给出一个主方案、一个降级方案和两个淘汰方案；
- 输出 D0-D10 构建计划、架构图、指标、风险和 Beta 升级路径。
```

## 3. RAG/知识库专项 Prompt

```text
请为以下企业知识库 PoC 选型：
- 文档：{类型、数量、页数、语言、扫描件/表格比例}
- 问题：{典型查询}
- 权限：{文档级/部门级/用户级}
- 指标：{Recall@K、引用正确率、答案正确率、拒答率、时延}
- 部署：{填写}
- 周期/团队：{填写}

必须实际比较：
- Dify / RAGFlow / FastGPT / MaxKB / 轻量自研（按当期维护状态调整）
- 解析/OCR：平台内置与 MinerU/Docling/PaddleOCR 等当期候选
- 检索：pgvector / Elasticsearch/OpenSearch / Milvus/Qdrant（只在适用时）
- 模型：国内获批 API 与自部署路径
- 评估：业务 Golden Dataset + Ragas/DeepEval/平台追踪等
- Vibe Coding：仅用于必要的 Adapter、权限与 UI 定制

先设计 20-50 个问题的小样本 bake-off，不能在解析和召回未验证前决定平台。
输出唯一主方案、降级方案、D3 切换条件与 10 日计划。
```

## 4. Agent/工作流专项 Prompt

```text
请为以下 Agent/工作流 PoC 选型：
{粘贴业务流程}

先回答：
1. 哪些步骤必须确定性执行？
2. 哪些步骤确实需要模型判断？
3. 是否需要持久状态、循环、分支、恢复、人审？
4. 工具调用失败如何重试、回滚和升级人工？
5. MCP 是必要标准化接口还是额外复杂度？

比较：
- 普通代码/规则工作流
- Dify/n8n/Coze Studio 等可视化路径
- 轻量模型 SDK/PydanticAI 类路径
- LangGraph 类状态图路径

非必要不得推荐多 Agent。最终输出工具权限表、状态机、失败路径、
评估集、主方案、降级方案和 Beta 升级路径。
```

## 5. 私有化/信创专项 Prompt

```text
请在数据不出域/信创约束下完成 PoC 技术选型：
- 服务器/加速卡：{填写}
- OS/容器/K8s：{填写}
- 网络：{离线/代理/白名单}
- 模型来源与许可证：{填写}
- 数据等级：{填写}
- 目标并发/时延：{填写}

必须联网核验：
- 目标模型与硬件的官方兼容矩阵
- vLLM/SGLang/LMDeploy/Xinference/MindIE/MindSpore 等当期适配
- RAG/平台依赖镜像与离线安装要求
- Embedding/Reranker 的本地可用性
- 国产数据库、网关、对象存储和观测系统兼容

输出：
- 最小单机 PoC 方案
- 若当前硬件不满足的替代 API/模型/量化方案
- 离线包与镜像清单
- 性能基线脚本与采集指标
- PoC 到生产容量规划中必须重做的部分
```

## 6. Vibe Coding 专项 Prompt

```text
请为该 PoC 设计安全的 Vibe Coding 开发方案。

必须搜索当前可用的：
- 国际 IDE/CLI：Cursor、Claude Code、Codex 等
- 国内工具：Trae、CodeBuddy、Qoder、通义灵码、Comate 等
- 开源/自托管方向：Cline、Roo Code、OpenCode、Aider 等

对比：
- 目标语言与仓库理解能力
- 企业数据保留/训练/地域策略
- 自定义模型或国内模型支持
- MCP/终端/测试/代码审查能力
- 团队协作、审计、私有化与价格

输出：
1. 唯一推荐工具组合；
2. 未批准数据不得进入工具的规则；
3. requirements.md / architecture.md / tasks.md；
4. 分支、依赖锁、测试、Review 与安全扫描；
5. 哪些代码允许生成，哪些必须人工编写/审核；
6. PoC 结束后的保留/重写/删除清单。
```

## 7. 技术刷新 Prompt

```text
今天是 {自动读取当前日期}。
请刷新本 Skill 的技术候选，不修改方法论：

1. 搜索过去 180 天内模型、RAG/Agent 平台、推理、评估、MCP、
   Vibe Coding 和国内云产品的重要变化；
2. 只使用可核验来源，优先官方 Release；
3. 标记新增、重大变化、弃用、许可证变化和维护停滞；
4. 对 references.md 提出更新建议；
5. 不因版本更新自动改变默认选型原则。
```
