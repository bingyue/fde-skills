# FDE-PoC-Tech-Stack-Selector — 执行清单

## 1. 进入门禁

- [ ] 场景卡或等价需求已经确认
- [ ] PoC 只有一个 Primary Metric
- [ ] 核心假设可在当前周期证伪
- [ ] 范围内/范围外已书面化
- [ ] AIBP/业务 Owner 确认业务真值
- [ ] IT/安全确认数据等级与部署边界
- [ ] 已明确 PoC 周期、人员和预算
- [ ] 若以上不满足，先回到 Discovery/SOW，不开始工具选型

## 2. 约束收集

- [ ] 数据格式、规模、语言、质量和敏感性
- [ ] 是否允许出域、进入第三方模型或 AI 编码工具
- [ ] 公有云/私有化/离线/混合云
- [ ] CPU/GPU/NPU、型号、显存、驱动和容器环境
- [ ] 信创、等保、行业监管和采购约束
- [ ] 并发、时延、吞吐、可用性和成本边界
- [ ] 必须集成的 API、数据库、SSO、IM 和审批
- [ ] 团队语言、框架、运维和 AI 经验
- [ ] PoC 成功后的 Beta/生产预期

## 3. 强制最新检索

- [ ] 报告记录执行日期与搜索日期
- [ ] 有 ≥8 个有效来源
- [ ] 有 ≥4 个官方文档/仓库/Release
- [ ] 有 ≥3 个国内官方来源或国内实践
- [ ] 最新结论优先引用 180 天内来源
- [ ] 模型/API 已核验地区、数据策略和当期能力
- [ ] 开源平台已核验最新 Release、许可证和维护状态
- [ ] 推理框架已核验目标模型与硬件兼容
- [ ] Vibe Coding 工具已核验企业数据政策
- [ ] 检索了至少一个同类国内落地案例
- [ ] 聚合文章未作为关键结论唯一来源
- [ ] 无联网能力时已停止最终推荐

## 4. 路径判断

- [ ] 已判断 L1 开箱即用是否足够
- [ ] 已判断 L2 Vibe Coding 定制是否必要
- [ ] 已判断 L3 工程化触发条件是否真实存在
- [ ] 选择的是最低可行复杂度
- [ ] 能用确定性流程时没有强行使用 Agent
- [ ] 能用 API 验证时没有无依据自部署模型
- [ ] PoC 未提前引入非必要 Kubernetes/微服务/多 Agent

## 5. 候选与硬门禁

每个候选均检查：

- [ ] 数据和代码流向可获批
- [ ] 许可证和商业使用边界清晰
- [ ] 目标网络、硬件和部署兼容
- [ ] 能在 PoC 周期验证 Primary Metric
- [ ] 团队能够调试、接管和回退
- [ ] 无不可接受的停止维护或供应商风险

## 6. 技术栈完整性

- [ ] 模型/API：选择、Adapter 与备选
- [ ] Prompt/结构化输出：版本化方式
- [ ] 编排：普通代码/工作流/Agent 的理由
- [ ] RAG/文档：解析、切分、Embedding、检索、Rerank
- [ ] 数据：数据库、向量库、缓存、对象存储
- [ ] UI/API：演示入口与集成接口
- [ ] 工具/MCP：权限、Schema、失败策略
- [ ] 评估：Golden Dataset、回归与业务验收
- [ ] 可观测：Trace、时延、Token/成本、失败分类
- [ ] 部署：环境、镜像、密钥、日志、备份
- [ ] Vibe Coding：工具、数据边界、测试和 Review

## 7. 明确决策

- [ ] 只有一个主方案
- [ ] 主方案逐层写了具体技术
- [ ] 每个技术都有“为什么适合本场景”
- [ ] 有一个 1–2 天内可切换的降级方案
- [ ] 降级触发条件有日期/指标
- [ ] 至少两个淘汰方案有明确理由
- [ ] 没有用 Star 数或“行业最强”代替判断
- [ ] 评分表与最终结论一致；不一致时解释硬约束

## 8. Vibe Coding 安全门禁

- [ ] 已生成 `requirements.md`
- [ ] 已生成 `architecture.md`
- [ ] 已生成 `tasks.md`
- [ ] 已准备验收测试/Golden Dataset
- [ ] 使用独立分支和最小权限
- [ ] `.env.example` 不含真实密钥
- [ ] 依赖已锁版本并检查许可证/漏洞
- [ ] 客户代码和数据只进入获批工具
- [ ] 权限、SQL、文件、网络和删除操作人工评审
- [ ] AI 生成代码通过单元测试和人工 Review
- [ ] AI 无权自行批准合并或部署
- [ ] PoC 结束有保留/重写/删除清单

## 9. 10 日构建与验收

- [ ] D0 场景/数据/安全双确认
- [ ] D1 完成检索、bake-off 和 TDR
- [ ] D2 最小架构可运行
- [ ] D3–4 核心链路跑通
- [ ] D5 Golden Dataset 基线完成
- [ ] D6–7 Primary Metric 有针对性改进
- [ ] D8 用户流程与人工兜底验证
- [ ] D9 回归、降级、成本和风险完成
- [ ] D10 Go/No-Go 书面评审

## 10. PoC→Beta/生产

- [ ] 明确保留的 PoC 资产
- [ ] 明确必须重写的代码和配置
- [ ] 补充 RBAC/SSO/租户/审计要求
- [ ] 补充容量、SLA、备份、灾备和运维
- [ ] 补充模型网关、限流、成本和内容安全
- [ ] 补充数据治理、评估回归和采纳看板
- [ ] 低代码平台的生产许可和迁移路径已确认
- [ ] Go/No-Go/再实验条件均可量化

## 11. 最终质量

- [ ] 对照 `evaluation.md` 总分 ≥80
- [ ] 无红线项
- [ ] `tech-decision-record.md` 已填写
- [ ] `poc-build-plan.md` 已填写
- [ ] AIBP、FDE、IT/安全完成评审
