# Diagnostic-FDE — Checklist

## 启动

- [ ] 行业、客户名、初始需求已确认
- [ ] 执行范围明确（全流程 / 单步）
- [ ] 已有材料已盘点，跳过项已标注

## Phase 1 情报收集

- [ ] ① 行业速览：≥3 来源，四维质量评分，价值链≥4 环节
- [ ] ② 竞品分析：≥2 竞品，12 维评分+证据，SWOT 完整
- [ ] ③ 客户诊断：假设≥10 条，3 类画像，IT 现状四维评估

## Phase 2 需求锁定

- [ ] ④ Issue Tree ≥3 层，MECE 检查通过，RICE 排序完整
- [ ] ⑤ 能力匹配：全部需求已匹配，✅ 率 ≥ 60%
- [ ] ⑤ 所有 ❌ 项有缓解策略，≥2 个权衡决策有 Pros/Cons

## Phase 3 方案设计

- [ ] ⑥ 战略类型有依据，OKR 级联≥2 层，分 3 阶段
- [ ] ⑦ 系统/数据流/部署 3 张 Mermaid 可渲染
- [ ] ⑧ ROI 至少 2 类测算，含敏感性分析三档

## Phase 4 包装输出

- [ ] ⑨ Why Us：≥3 差异化维度，坦诚劣势，≥1 同行业案例
- [ ] ⑩ POC：范围明确，指标≥3，时间线到周
- [ ] ⑪ 执行摘要≤1 页，PPT≤15 页，数字可追溯
- [ ] ⑫ 内部评审全部勾选

## 提交前

- [ ] 所有文档中文，架构图英文标签+中文注释
- [ ] 敏感信息已脱敏
- [ ] 客户授权范围已确认
- [ ] POC 方案含 Beta/生产路径、私有化/Gateway 约束
- [ ] 执行摘要含 AIBP 分工与采纳运营建议
