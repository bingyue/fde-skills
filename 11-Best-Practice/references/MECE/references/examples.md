# MECE 使用示例

按需加载，非执行必需。

## 示例1：分析类问题——行业竞争格局

```
用户：分析一下新能源汽车行业的竞争格局

一、新能源汽车行业竞争格局：头部集中度提升，价格战加速出清

二、市场格局
- 第一梯队（市占率>10%）：比亚迪、特斯拉，合计占45%份额
- 第二梯队（市占率5-10%）：蔚小理、极氪，差异化定位
- 第三梯队（市占率<5%）：传统车企新能源品牌，承压明显

三、竞争焦点
- 价格维度：2024年行业平均降价12%，尾部品牌毛利已转负
- 技术维度：800V高压+城市NOA成为20万以上车型标配
- 渠道维度：直营vs经销模式之争，新势力倾向直营

四、趋势判断
- 短期：价格战持续，2-3家尾部品牌可能出局
- 中期：固态电池量产将重塑技术壁垒
- 长期：出海能力决定天花板
```

✅ 结论先行、MECE分组（梯队/维度/时间）、每层有观点。

## 示例2：总结类问题——梳理报告核心观点

```
用户：梳理这几份研报关于AI算力需求的核心观点

一、AI算力需求：训练端增速放缓，推理端需求爆发，2025年推理占比将超70%

二、训练端：从"越大越好"转向效率优化
- 模型参数增速放缓：GPT-4级模型已接近收益递减拐点
- 训练方法改进：数据质量>数据量，MoE架构降低训练成本40%

三、推理端：应用爆发驱动算力需求指数增长
- 用户规模扩张：ChatGPT周活用户突破2亿，每用户推理成本是搜索10倍
- 多模态推理：图像/视频推理算力需求是文本的100倍
- Agent模式：单次任务需5-20次模型调用，推理需求倍增

四、投资含义
- 算力基础设施：推理侧芯片（推理卡/边缘计算）需求增速高于训练卡
- 能源配套：AI数据中心功耗密度是传统的5-8倍，电力基础设施成瓶颈
```

✅ 从杂乱信息中提炼结构、每组MECE、概括有观点非贴标签。

## 示例3：论证类问题——投资决策建议

```
用户：为什么我们认为应该增持XX公司？

一、建议增持XX公司，目标价XX元，上行空间30%

二、增持理由（归纳法，3个独立论据）
- 估值修复空间大：当前PE处于近5年20%分位，低于行业平均
- 基本面拐点确认：Q3营收同比转正，新订单增长50%+
- 催化剂临近：XX政策预计Q1落地，直接受益

三、风险提示
- 核心风险：XX原材料价格波动（若涨超20%将侵蚀毛利3个百分点）
- 应对策略：已通过期货对冲50%敞口
```

✅ 关键句层用归纳（3个独立理由）、支撑层用演绎/数据论证。
