# Elite PPT Pro — QA 检查清单

每次生成PPT之前，**必须按顺序通过所有检查**。

---

## 阶段一：研究阶段检查（Step 1完成后）

- [ ] 已收集 ≥15 个具体数据点（带数字，非定性描述）
- [ ] 数据来自 ≥5 个不同来源
- [ ] 每张内容页均有对应数据支撑
- [ ] 至少1组历史趋势数据（用于图表）
- [ ] 竞争对比数据已收集
- [ ] 所有引用已记录：来源机构名称 + 发布日期

---

## 阶段二：内容质量检查

### 标题规范
- [ ] 每张幻灯片标题写**结论**，不写话题
  - ✅ `"AI市场集中度持续上升，头部三强吃掉65%份额"`
  - ❌ `"AI市场分析"`
- [ ] 封面副标题包含核心洞察数字
- [ ] 目录（如有）的每项描述具体内容而非泛化章节名

### 数据来源标注
- [ ] 每张含数据的幻灯片底部有 `Source:` 标注
- [ ] Source格式：`Source: [机构名], [机构名] | [时间段]`
- [ ] 数据不超过2年旧（快速变化行业不超过1年）
- [ ] 无不可核实的孤立声明（"显著增长"等模糊说法）

### 信息密度
- [ ] 每张内容页至少4个内容区域
- [ ] 图表有图表标题
- [ ] 每页关键数字/结论有视觉强调（粗体或强调色）

---

## 阶段三：视觉规范检查

### 颜色规范
- [ ] 全文每页最多2个强调色（多系列图表内部除外）
- [ ] 颜色为6字符hex，无#前缀（`"00A69C"` ✓，`"#00A69C"` ✗）
- [ ] 主题配色统一，未混入其他风格颜色

### 布局规范
- [ ] 所有内容页顶部有标题条（高度 0.55"，白底，底部细线）
- [ ] 所有非封面页右下角有页码徽章
- [ ] 内容页背景为白色
- [ ] 没有元素超出幻灯片边界（x+w ≤ 10, y+h ≤ 5.625）

### 排版规范
- [ ] 中文字体：Microsoft YaHei；英文字体：Arial
- [ ] 标题字号 ≥ 10pt，正文字号 ≥ 8pt
- [ ] 所有多行文本设置 `wrap: true`
- [ ] 文字框高度足够（不截断内容）

---

## 阶段四：技术规范检查

- [ ] 形状使用字符串形式（`"rect"`/`"oval"` 而非 `pres.ShapeType.xxx`）
- [ ] 渐变填充格式正确：`{ type:"gradient", color:["起始色","结束色"], angle:90 }`
- [ ] `pres.writeFile({ fileName: 'xxx.pptx' })` 语法正确
- [ ] 所有 `createSlide` 函数为同步函数（非async）
- [ ] 文件编码 UTF-8

---

## 常见错误速查

| 错误现象 | 原因 | 解决方案 |
|----------|------|---------|
| 颜色不生效 | 颜色带#前缀 | 去掉#，只留6字符 |
| 文字重叠/截断 | 未设wrap或h太小 | 加 `wrap:true` + 增大h |
| 形状不渲染 | `pres.ShapeType.xxx`不存在 | 改用字符串 `"rect"` |
| 渐变不显示 | stops格式错误 | 用 `color:["色1","色2"]` |
| 图表为空 | labels和values长度不一致 | 检查数组长度 |
| 中文乱码 | 字体设置错误 | 改为 "Microsoft YaHei" |
| 元素被裁切 | 坐标超出边界 | 确保 x+w ≤ 10, y+h ≤ 5.625 |
| 数据声明无来源 | 跳过了研究阶段 | 返回Step 1补充来源 |

## 标准咨询报告结构

| 页码 | 页面类型 | 说明 |
|------|----------|------|
| 1 | 封面 Cover | 含核心洞察数字的副标题 |
| 2 | 核心摘要 | 3-4个最重要结论（带数据） |
| 3-4 | 市场背景 | 规模、增速、趋势图 |
| 5-6 | 竞争分析 | 格局、市占率、差异化 |
| 7-8 | 深度分析 | 框架模型（SWOT/波特等） |
| 9 | 战略建议 | 3-5条可执行建议 |
| 10 | 附录/来源 | 完整数据来源列表 |

**典型页数：8-15页**
