# FDE-Full-Lifecycle — Evaluation

## 验收指标

| 指标 | 门槛 | 说明 |
| --- | --- | --- |
| 审计完整度 | ≥ 80% 清单项完成 | 对照 fde-framework 审计清单 |
| 痛点排序共识 | 业务方签字 Top 3 | 避免技术自嗨 |
| POC 验证 | 核心假设可验证/否证 | 有明确通过/不通过标准 |
| ROI 可追溯 | 假设有来源标注 | 保守/基准/乐观至少两档 |
| 部署可用性 | UAT 通过 | 真实用户参与 |
| 私有化/信创 | Gateway+RBAC 过门禁 | 数据不出域、审计齐全 |
| 采纳衔接 | Beta 培训/看板计划 | 技术验收≠业务成功 |
| AIBP 共签 | Ground Truth 与验收确认 | FDE 独自定义指标 |
| 抽象沉淀 | ≥ 1 项可复用资产 | 组件/模板/行业 Know-how |

## 通过标准

- 三阶段均有书面交付物
- 客户或业务 Owner 确认价值目标
- 无未闭环的高风险项

## 失败信号

- 跳过 Audit 直接开发
- POC 无验收指标
- 项目结束仅交付 Demo 链接，无运维与抽象

## 验收样例

**通过**：制造企业客服场景，审计识别 3 个痛点，POC 验证 RAG 首响时间降 40%，UAT 通过，沉淀 FAQ 评估集模板。

**不通过**：仅完成 Streamlit Demo，无干系人访谈记录，客户未确认验收标准。
