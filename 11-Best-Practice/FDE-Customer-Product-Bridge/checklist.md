# FDE-Customer-Product-Bridge — Checklist

## 启动

- [ ] 工作流类型已确认（A/B/C/D）
- [ ] 项目阶段已确认（PoC/Beta/生产）
- [ ] 输出受众已明确
- [ ] 客户授权与脱敏要求已确认
- [ ] AIBP 协作边界已对齐（见 AIBP-Collaboration-Playbook）

## 工作流 A：调研

- [ ] 六段访谈提纲已准备
- [ ] 业务流程表完整（环节/角色/输入/输出/频次）
- [ ] 痛点有业务影响与量化线索
- [ ] 产品/研发输入表已填写
- [ ] 待验证问题已列出
- [ ] PoC 假设与 Beta 采纳路径已标注

## 工作流 B：Demo

- [ ] Demo 目标与听众角色矩阵完成
- [ ] 故事线：场景→痛点→路径→结果→价值
- [ ] 每演示步骤对应一个痛点
- [ ] 异议回应表已准备
- [ ] 网络/数据/权限/信创备用方案已准备
- [ ] Demo 后推进项含周度 Demo 节奏

## 工作流 C：案例

- [ ] 五条件判断已通过
- [ ] 可复制条件与不适用边界已写明
- [ ] 结果指标有证据状态标注
- [ ] 分角色讲解版本已输出
- [ ] 阶段标注（Beta/生产）与脱敏完成

## 工作流 D：反馈转化

- [ ] 六维判断已完成
- [ ] 用户故事格式正确
- [ ] 必须支持 / 暂不支持 / 依赖 / 风险 已界定
- [ ] 异常路径与验收标准已填写
- [ ] 生产项含 RBAC/审计要求（如适用）

## FDE 阶段门禁

- [ ] PoC：调研/Demo 输出可支撑场景卡双签
- [ ] Beta：案例/反馈含采纳与培训建议
- [ ] 生产：反馈单可过 PRD-Generator / RBAC-Audit 评审

## 提交前质量自检

- [ ] 还原了业务流程，非仅列功能
- [ ] 区分客户原话、事实、判断、待验证
- [ ] 产品侧有场景、价值、复用性、优先级
- [ ] 研发侧有输入、输出、边界、异常、验收
- [ ] 无未经授权的商业承诺
