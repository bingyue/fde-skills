# FDE-Customer-Product-Bridge — Evaluation

## 验收指标

| 工作流 | 指标 | 门槛 |
| --- | --- | --- |
| A 调研 | 流程环节覆盖度 | 主流程 100% 覆盖 |
| A 调研 | 产品/研发可执行输入 | ≥ 3 条有验收标准 |
| A 调研 | PoC/Beta 衔接 | 含采纳或培训建议 ≥ 1 条 |
| B Demo | 痛点-步骤对应率 | 100% 步骤有对应痛点 |
| B Demo | 备用方案完备 | 3 类异常均有预案（含私有化/信创） |
| C 案例 | 可复制条件明确 | 适用/不适用均写明 |
| D 反馈 | 验收标准可测试 | 研发可据此写用例 |
| 通用 | 事实/判断分离 | 无混用 |
| 通用 | AIBP 确认项 | 关键口径已标注待 AIBP 签字 |

## 通过标准

- 目标受众能直接使用输出物，无需二次大幅改写
- 产品/研发反馈单可直接进入 backlog 评审
- Demo 脚本可在 30 分钟内彩排完成
- PoC 输出可衔接 Consultative-Problem-Solving / PRD-Generator

## 失败信号

- 调研纪要只有功能列表无流程
- Demo 按产品菜单讲解
- 案例无数据支撑或无可复制边界
- 反馈单缺少异常路径
- 无阶段标注（PoC/Beta/生产）

## FDE 阶段对齐

| 阶段 | 额外验收 |
| --- | --- |
| PoC | 输出支撑场景卡与 SOW 范围 |
| Beta | 含 FDE-Adoption-Growth 可执行建议 |
| 生产 | 反馈单含 RBAC/审计/Runbook 引用 |

## 验收样例

**通过**：零售客户 PoC 调研纪要含 8 环节流程表，3 条研发输入带验收标准，Demo 脚本 5 步均映射痛点，标注 Beta 培训计划。

**不通过**：将客户「要智能客服」原话直接作为 PRD 标题，无流程还原，无 AIBP 确认项。
